1. KISS

Keep It Simple Stupid
Django is not an MVC framework, it is nothing more than routes & views; a simple and easy to use framework. Everyting else is optional.

2. Essence of a Framework

A framework should essentially:

  • Protect you from the dangerous things
  • Protect you from the boring things

3. DNIY

Don't Need It Yet
If you're wondering whether you need class based views, you probably don't need them, just use a function based view. Use url shortcuts. The same applies for most areas of software development: You don't need REST to do simple things like login

4. Don't reinvent the wheel

There are a myriad of 3rd party applications for Django, go forth and explore! If you need a friendly push, here are some good ones you may find helpful:

  2. django-allauth
